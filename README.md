# Projeto SQL – Sistema de Gestão de Cafeteria

Neste projeto, eu desenvolvi um banco de dados completo para simular o funcionamento de uma cafeteria, organizando informações de forma estruturada e estratégica.
Modelei e implementei tabelas relacionadas a:

Clientes

Fornecedores

Funcionários

Pedidos

Itens dos pedidos

O foco foi trabalhar bem os relacionamentos entre as entidades, garantindo integridade referencial com uso de foreign keys e regras como ON DELETE CASCADE.
Além da modelagem, explorei diferentes níveis de consulta:

Consultas simples com SELECT, WHERE e ORDER BY

Consultas com JOIN

Agrupamentos com GROUP BY

Subconsultas mais elaboradas
Criação de Views para organizar e reaproveitar consultas estratégicas
Implementação de Triggers para automatizar regras de negócio dentro do banco
A ideia desse projeto foi ir além do básico e estruturar algo mais próximo de um cenário real, aplicando boas práticas de organização, separação de scripts (estrutura, dados e consultas) e clareza na modelagem.
Tive o intuito também de me testar e buscar em mim a expertise de resolver ou explorar demandas que ainda estou em fase de conhecimento. Isso ampliou em muito a minha formação pessoal, minha curiosidade em tentativa e erro e minha busca por aprimoramento.

Esse projeto representa meu desenvolvimento em SQL e meu interesse em entender não apenas comandos isolados, mas a arquitetura e o funcionamento lógico de um banco de dados bem estruturado.
